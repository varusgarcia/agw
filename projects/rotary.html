Wireless Rotary Phone

First off, this project was done for a course at my university. The task was to fork or make a version of an object, technique, project, etc.. It had to make an alternative version of whatever we chose and give it another direction. I just wanted to mention this to give you some context, otherwise my project might not make little sense.
I chose to make a version of an rotary phone, you now, those our parents used. My version would work wireless. This project might not make sense but it can show you how fast you can work with arduino, going from prototyping to the actual finished product in less than a week.

What you need:

Old Rotary Phone

The phone has to be in good conditions, as we will use some of the parts. The rotary dialer and the ringer mechanism have to still work.

Arduino

I used an Arduino Mega because it has so many digital pins, so that you can connect all the cables directly to the arduino and don't need any other accesory.

Mobile Phone

Here i used an old Samsung phone i had laying around. It's important that the phone has physical buttons (because we will later hack those) and that it still works.

20x 220 Ohm Resistor

These we need for the Optocoupler.

4x Optocoupler

These we need to "press" the phone buttons. The ones i used were the ACPL-847-000E.

H-Bridge

This one we need for using the ringer mechanism

Computer and Arduino Software

You will need a computer with the Arduino software installed

What can the Phone do?

The rotary phone can do the basic tasks of an mobile phone. Receive and make calls.
It can be expanded to send sms and basically everything a mobile phone can do, but this isn't my prirority.

How it works

First, let's see how everything is wired:

The dialing mechanism

Let's start with the dialing mechanism. The condition to dial is that the phone handset has to be taken. As you can see below, there is a simple switch that opens when the headset is taken.

Then, the arduino knows that someone wats to input a number. As you can see below, it works really easy, there are two circuits that open (when the wire is grey) and close (when the wire is red) in a specific way for each number.
One of them, closes while the number is beeing dialed, en the other one closes and opens for each number (in this case 10 times).

Ok, now the arduino has the information about the number it has to dial on the phone.
In the image below you can see how each number on the phone is wired to the optocoupler, then to the resistor and finally to the arduino board.
Basically each pin on the Arduino controls one number on the phone:

Pin 23 39 41 45 33 35 37 29 29 31 43 47

WTF is an Optocoupler and why do I use it?

An Optocoupler has two different circuit inside, if the first one is HIGH, the second one closes. It works basically like a transistor, with the difference thet the two circuits are seprated.
The first circuit has an Led inside (that's why we need to put a resistor before) The second one basically has a light detector, that has the property of letting current pass through if it detects light.
So why have i used this instead of a normal transistor? Well, the thing with transistors is that the two circuits would need to have the same current. In my case they don't (arduino run on 5v and the Phone on 3.3v)

When that pin is set to HIGH (5v are coming out of that pin), the optocoupler closes the circuit and the number is "pressed" on the phone.

Each time a new number is pressed, the arduino starts a countdown of about 5 sec., If no new number is dialed in that time, it "presses" the call button and the phone calls the number.
